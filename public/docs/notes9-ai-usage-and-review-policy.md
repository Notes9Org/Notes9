# Notes9 AI Usage and Review Policy

## Purpose

This document defines how teams should use AI-enabled features in Notes9 responsibly. It is written for scientific and operational settings where speed is useful but accuracy, provenance, and review remain mandatory.

## Policy Statement

AI assistance in Notes9 is an accelerator for drafting, retrieval, and synthesis. It is not an independent decision-maker and must not be treated as authoritative without human review.

## Approved Use Cases

AI features may be used for:

- summarizing selected text
- rewriting content for clarity
- drafting note structure
- surfacing candidate citations
- helping users navigate stored context
- generating first-pass reporting language

These use cases are appropriate when a qualified person reviews the output before it is treated as part of the working record.

## Restricted Use Cases

AI features should not be used without explicit human validation for:

- final scientific claims
- regulatory or ethics statements
- protocol-critical instructions
- patient-impacting or clinical interpretation
- irreversible operational decisions

## Required Reviewer Behavior

Any person using AI output in a project record should:

- verify factual statements
- verify citation relevance
- remove unsupported claims
- correct language that overstates certainty
- preserve the connection to the source context

If the reviewer cannot verify the output, it should not be adopted.

## Citation Guidance

When using features such as Cite with AI:

1. select the exact sentence or claim that needs support
2. review the returned citation candidates
3. choose the source that actually supports the claim
4. confirm author, title, year, and relevance
5. insert bibliography entries only after review

Citation tools speed formatting and discovery, but they do not remove the need for source judgment.

## Documentation Expectations

If AI materially influences a note, summary, or draft, the user should ensure:

- the surrounding record still reflects actual human intent
- citations remain traceable
- ambiguous phrasing is resolved before sharing
- the final saved content is suitable for future retrieval

## Governance Controls

Workspace owners should define:

- who may use AI features
- what review standard applies
- where AI-assisted drafting is acceptable
- what documentation must remain fully manual

Teams should align this policy with their internal governance, quality, and compliance requirements.

## Red Flags

Stop and review immediately if users begin to:

- accept AI output without checking sources
- insert bibliography entries they have not reviewed
- use generated text to replace actual observations
- blur the difference between evidence and suggestion

## Minimum Safe Standard

For most teams, a practical minimum standard is:

- AI may draft
- humans must verify
- provenance must remain intact
- final judgment stays with the researcher or reviewer

## Implementation Recommendation

During onboarding, show one approved example and one unacceptable example of AI-assisted usage. Teams adopt the policy faster when the standard is concrete instead of abstract.
